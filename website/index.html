Core capability map

Built for operators who need real infra context, not a generic asset list.

Rackpad covers the day-to-day modeling work of a modern lab: physical rack placement, switch and patch panel ports, WiFi topology, discovery staging, VLAN-aware IPAM, virtual guest capacity, user roles, and monitoring with alert delivery.

Racks and devices

Physical placement with room, shelf, wireless, and virtual spillover

Track named racks like NET-01 and CMP-01, but also loose room tech, rack shelves and trays, blanking panels, brush panels, hosted VMs, and wireless clients without losing relationships.

Ports and cabling

Structured ports with access, trunk, bridge, and VM context

Model RJ45 and SFP ports, access/native/tagged VLAN behavior, patch cables, peer links, reusable port templates, and hypervisor-side virtual switch memberships for servers, switches, firewalls, AP uplinks, and VMs.

VLANs and IPAM

Subnet-aware inventory and address allocation

Create VLAN ranges, assign subnets, define DHCP scopes and IP zones, and keep management addressing synchronized with the devices that actually consume it.

Compute

Hosts, guests, virtual switches, and documented capacity

Link VMs to hosts, record CPU, memory, storage, and human specs text, map host uplinks and guest NICs onto shared virtual bridges, then compare the host footprint against the guests assigned to it.

WiFi

Controllers, SSIDs, radios, and client associations

Capture APs, controllers, SSIDs, radios, channels, bands, client telemetry, and last-roam context instead of pretending WiFi is just another unmanaged endpoint list.

Monitoring and alerts

Multi-target health checks with notification delivery

Run separate management, storage, service, or VIP checks per device and send alerts to Discord, Telegram, or SMTP email when a target goes down or recovers.

Deployment

Fastest path: Linux + Docker + GitHub clone.

Rackpad is easiest to test as a Docker deployment on a Linux server or VM. The static product site you are reading is separate and IIS-friendly; the app itself remains the Docker or Node service.

Minimum test host
2 vCPU | 4 GB RAM | 10 GB free disk

Practical starting point for a single-user lab deployment with SQLite, Docker, and moderate demo data.

Recommended
4 vCPU | 8 GB RAM | 20+ GB SSD

Better fit if discovery, monitoring, backups, and a larger real lab inventory will all be active together.

Platform
64-bit Linux with Docker Engine and Compose

Ubuntu or Debian-class hosts are the easiest path for first deployment and update cycles.

What gets deployed?

Rackpad ships as a React frontend, Fastify API, and SQLite persistence path in one containerized app. Docker gives you persistent storage, health checks, and an easier upgrade path without forcing you to manage a separate database stack.

Quick start from GitHub clone, configure, and start Rackpad 
git clone https://github.com/Kobii-git/Rackpad.git
cd Rackpad
cp .env.example .env
docker compose up --build -d

# then open:
# http://SERVER_IP:3000
From the first browser session

Rackpad is ready to move from empty bootstrap to a full lab model.

On first run, the app creates the initial admin account and lets you choose an empty environment or a richer demo environment that already contains multiple labs, racks, WiFi controllers, guest VMs, discovery states, and monitoring targets.

1Bootstrap the admin

Create the first operator account, sign in, and decide whether to start empty or preload the demo environment.

2Document the physical layer

Add racks, room gear, servers, switches, patch panels, power gear, and device templates before you start linking ports and cables.

3Connect the network context

Capture VLANs, subnets, DHCP scopes, IP zones, trunk and access port behavior, and management addressing.

4Operate and review

Turn on monitoring, stage discovery results, configure alerts, manage users, and keep backups in the admin area.
