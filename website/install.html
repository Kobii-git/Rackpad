Requirements

Use practical operator requirements, not theoretical minimums.

Rackpad is light compared with large DCIM suites, but it still benefits from a stable Linux host, persistent storage, and a runtime that can support SQLite, monitoring checks, and regular backups.

Area Recommended starting point Notes
Host OS 64-bit Linux VM or server Ubuntu or Debian-class hosts are the easiest path for Docker and Node-based testing.
CPU and RAM 2 vCPU / 4 GB RAM minimum, 4 vCPU / 8 GB recommended Enough for the app, SQLite, monitoring, and a moderate lab inventory without contention.
Disk 10 GB free minimum, 20+ GB SSD preferred Allows room for images, rebuilds, backup exports, and growth in audit and discovery data.
Networking Port 3000 reachable from the admin network Use a reverse proxy and TLS if the app leaves a private LAN or VPN-only environment.
Operational reach ICMP and TCP reachability from the Rackpad host or container Discovery, monitoring, and some WiFi or bridge workflows are only useful if Rackpad can actually reach the subnets, controllers, and host uplinks you want it to observe.
Alert egress Outbound SMTP, Discord webhook, or Telegram API access if notifications are enabled Optional, but needed for alert delivery beyond the in-app monitoring workspace.
Runtime Docker Engine + Compose plugin, or Node 22 LTS + npm Docker is the recommended test path. Native Node is best for custom service deployments.
Recommended path

Option 1: Docker on a Linux host

Docker is the fastest and most forgiving deployment path for Rackpad because it keeps the app runtime consistent, handles the bundled frontend/API container, and stores SQLite data in a persistent volume. The official Docker install guidance for Ubuntu is available from Docker Docs.

1Install Docker Engine and Compose

On Ubuntu or Debian, install Docker Engine from the official Docker repository, then confirm both the engine and compose plugin are available.

  • Install the Docker repository prerequisites and key.
  • Add the Docker apt repository for your Linux release.
  • Install Docker Engine, CLI, container runtime, buildx, and the Compose plugin.
  • Enable the Docker service and confirm it starts cleanly.
2Clone the Rackpad repository

Clone Rackpad from GitHub into a working directory such as /opt/Rackpad or your home folder.

Clone from GitHub private or public repository path 
git clone https://github.com/Kobii-git/Rackpad.git
cd Rackpad
3Create your environment file

Start with the bundled example file and set the port you want Rackpad to expose.

Basic environment setup minimum Docker path 
cp .env.example .env

# at minimum:
RACKPAD_PORT=3000

# if deploying behind a reverse proxy:
TRUST_PROXY=1
TRUSTED_HOSTS=rackpad.example.com
TRUSTED_ORIGINS=https://rackpad.example.com
4Build and start Rackpad

Compose builds the production image and starts the stack. The app will be reachable on the configured port once the container reports healthy.

Build and run one-container full app stack 
docker compose up --build -d
docker compose ps
docker compose logs -f --tail=100
5Complete the first-run flow

Open http://SERVER_IP:3000, create the initial admin account, sign in, and choose whether to start empty or preload the demo environment.

  • The demo path now includes racks, shelves or trays, brush and blanking panels, multi-target monitors, controller-aware WiFi, virtual switches, discovery entries, and compute guest mappings.
  • Use the demo first if you want a quick screenshot-ready walkthrough before documenting your real lab.
Official Docker references

For the current Docker Engine install flow on Ubuntu and supported distributions, verify the latest repository steps in Docker's documentation before rolling out broadly: docs.docker.com/installation/ubuntulinux

Alternative path

Option 2: Native Node.js install

Use the native path if you want Rackpad to run directly as a Linux service instead of inside Docker. This path expects Node 22 LTS and the ability to install the app dependencies locally.

1Install Node 22 and build prerequisites

Prepare the host with Node 22 LTS, npm, compiler tools, and Python so the SQLite dependency can install cleanly if a prebuilt binary is not available.

2Install app dependencies and build
Build for native service use Node 22 expected 
npm install
npm run build

HOST=0.0.0.0 PORT=3000 DATABASE_PATH=./rackpad.db npm start
3Optionally install as systemd service

Rackpad includes a sample rackpad.service file you can adapt for a native Linux service deployment. This is the better fit if you want Rackpad to reach monitoring or discovery targets directly from the host network instead of from a container.

Website hosting

Hosting the Rackpad product site on IIS

The marketing and documentation site in the website/ folder is a separate static bundle from the Rackpad app. It can be copied directly to an IIS site root such as rackpad.co.za.

  • Copy the contents of website/ to the IIS site root.
  • Keep the bundled web.config in place.
  • Enable the IIS Static Content and Default Document features.
  • Bind the site to HTTPS with your certificate.
  • Keep the app itself on Linux or Docker; IIS is only for the Rackpad product and documentation site.
