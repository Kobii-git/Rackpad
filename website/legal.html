Legal and release readiness

License, copyright, security, and support basics for Rackpad.

These notes are meant to make the project easier to publish, clone, deploy, and maintain before the V1 release. They do not replace professional legal advice, but they do give the repository the standard project files operators expect when evaluating a self-hosted tool.

Maintainer-facing note

Current assumption

For this release-prep pass, the repository license has been staged as MIT because there was no contrary preference supplied during implementation. If you want a different licensing model before V1, change the root license file before announcing the public release.

License

MIT for the repository source and docs

Rackpad is currently set up with an MIT license at the repo root. That gives future users a straightforward cloning and reuse story while keeping the standard copyright notice intact.

  • Source, docs, and website files are covered by the repo root license unless stated otherwise.
  • Third-party packages remain under their own licenses.
  • Redistributors should keep the Rackpad notice and third-party obligations intact.
Copyright and notice

Project identity and attribution

The project notice explains that Rackpad is a self-hosted inventory and operations tool and clarifies that the Rackpad name and visual identity can be used to refer to the project itself but should not be used to imply maintainer endorsement.

  • Copyright notice added at the repository root.
  • Website footer includes the same project identity language.
  • Third-party dependency references point back to package.json and package-lock.json.
Security

Private reporting path before public disclosure

Rackpad now includes a basic security policy so there is a private path for sensitive reports and a clear note that older releases may not receive backported fixes.

  • Prefer GitHub Security Advisories or private maintainer contact.
  • Do not open sensitive issues publicly first.
  • Deploy behind TLS and a reverse proxy before broader exposure.
Support

Clear expectations for a self-hosted operator tool

Rackpad is framed as a maintained self-hosted project, not a commercial managed service. The support note asks bug reports to include deployment method, version, browser context, and whether the issue reproduces against demo or restored data.

  • No implied SLA.
  • Best fit: homelabs, workshops, small offices, private infra inventories.
  • Release notes, install docs, and security guidance are linked from the repo root.
